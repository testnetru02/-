نظرًا لأننا نركز الآن بشكل كامل على القوالب الكتلية (Block Themes) منذ حوالي عام 2020، فلن نقوم بتحديث قالب البداية هذا _s (Underscores) بشكل نشط. سيتم أرشفة هذا المستودع للرجوع إليه فقط.

لمعرفة المزيد حول القوالب الكتلية، يرجى زيارة صفحة القوالب الكتلية ومتابعة المصادر المذكورة هناك.

_s

مرحباً. أنا قالب بداية يُدعى _s، أو Underscores إذا أردت. أنا قالب مُصمم ليكون قابلاً للتعديل البرمجي (hacking)، لذا لا تستخدمني كقالب أب (Parent Theme). بدلاً من ذلك، حاول تحويلي إلى أفضل وأروع قالب ووردبريس موجود. هذا ما خلقت من أجله.

قد تجعلني CSS البسيطة للغاية أبدو مثل "تارتار القوالب"، لكن هذا يعني وجود عناصر أقل تعترض طريقك أثناء تصميم قالبك الرائع. إليك بعض الأشياء الأخرى المثيرة للاهتمام التي ستجدها هنا:

    سير عمل حديث بواجهة أوامر جاهزة تجعل تجربة مشروعك أكثر متعة.

    كمية مناسبة من قوالب HTML5 الحديثة، النظيفة، والمعلّقة بشكل جيد.

    تنفيذ رأس ترويسة مخصص (custom header) في inc/custom-header.php. ما عليك سوى إضافة مقتطف الكود الموجود في التعليقات داخل inc/custom-header.php إلى ملف header.php الخاص بك.

    وسوم قوالب مخصصة في inc/template-tags.php تحافظ على قوالبك نظيفة ومرتبة وتمنع تكرار الكود.

    بعض التعديلات الصغيرة في inc/template-functions.php التي يمكن أن تحسّن تجربة تصميم القالب لديك.

    سكريبت في js/navigation.js يجعل قائمتك قابلة للطي على الشاشات الصغيرة (مثل هاتفك)، وجاهز للإبداع باستخدام CSS. تم إدراجه في functions.php.

    نموذجان للتخطيط في sass/layouts/ تم إنشاؤهما باستخدام CSS Grid لوضع شريط جانبي على أي من جانبي المحتوى. فقط قم بإلغاء تعليق التخطيط الذي تختاره في sass/style.scss. ملاحظة: يتم تحميل أنماط .no-sidebar تلقائياً.

    CSS starter منظم بذكاء في style.css سيساعدك على البدء بسرعة في تصميمك.

    دعم كامل لتكامل ملحق WooCommerce من خلال خطافات (hooks) في inc/woocommerce.php، وتجاوز أنماط woocommerce.css مع تمكين ميزات معرض المنتجات (تكبير، تمرير سريع، صندوق ضوئي).

    مرخص بموجب ترخيص GPLv2 أو أحدث. :) استخدمه لصنع شيء رائع.

بداية سريعة
قم باستنساخ أو تحميل هذا المستودع، غيّر اسمه إلى شيء آخر (مثلاً: megatherium-is-awesome)، ثم ستحتاج إلى القيام ببحث واستبدال مكون من ست خطوات للاسم في جميع القوالب.

    ابحث عن '_s' (داخل علامات تنصيص مفردة) للتعرف على نطاق النص (text domain) واستبدله بـ: 'megatherium-is-awesome'.

    ابحث عن _s_ للتعرف على جميع أسماء الدوال واستبدلها بـ: megatherium_is_awesome_.

    ابحث عن Text Domain: _s في ملف style.css واستبدله بـ: Text Domain: megatherium-is-awesome.

    ابحث عن _s (مع مسافة قبله) للتعرف على DocBlocks واستبدله بـ: Megatherium_is_Awesome.

    ابحث عن _s- للتعرف على المعرفات ذات البادئة (handles) واستبدلها بـ: megatherium-is-awesome-.

    ابحث عن _S_ (بأحرف كبيرة) للتعرف على الثوابت واستبدلها بـ: MEGATHERIUM_IS_AWESOME_.

بعد ذلك، قم بتحديث رأس ملف الأنماط في style.css، والروابط في footer.php بمعلوماتك الخاصة، وأعد تسمية _s.pot من مجلد languages لاستخدام اختصار القالب (slug). ثم قم بتحديث أو حذف ملف readme هذا.

الإعداد
لبدء استخدام جميع الأدوات التي تأتي مع _s، تحتاج إلى تثبيت تبعيات Node.js و Composer الضرورية:
bash

$ composer install
$ npm install

أوامر CLI المتاحة
_s يأتي محمّلاً بأوامر CLI مصممة خصيصاً لتطوير قوالب ووردبريس:

    composer lint:wpcs: يفحص جميع ملفات PHP وفقاً لمعايير ترميز PHP.

    composer lint:php: يفحص جميع ملفات PHP بحثاً عن أخطاء نحوية.

    composer make-pot: ينشئ ملف .pot في مجلد languages/.

    npm run compile:css: يترجم ملفات SASS إلى CSS.

    npm run compile:rtl: ينشئ ملف أنماط RTL.

    npm run watch: يراقب جميع ملفات SASS ويعيد ترجمتها إلى CSS عند تغييرها.

    npm run lint:scss: يفحص جميع ملفات SASS وفقاً لمعايير ترميز CSS.

    npm run lint:js: يفحص جميع ملفات JavaScript وفقاً لمعايير ترميز JavaScript.

    npm run bundle: ينشئ أرشيف .zip للتوزيع، مع استبعاد ملفات التطوير وملفات النظام.
